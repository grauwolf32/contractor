## 5. Cryptographic Functions
| Path | Function | Risk note |
|------|----------|-----------|
| `utils/jwt.py` | `jwt_auth_required` decorator — validates JWT from Authorization header, decodes token, injects user identity | **HIGH** — Uses `jwt.decode` with `options={"verify_signature": False}`; signature verification disabled, allowing token forgery |
| `utils/mock_methods.py` | `mock_jwt_auth_required` decorator — bypasses JWT validation entirely, injects hardcoded mock user | **CRITICAL** — If accidentally enabled in production, all protected endpoints become fully unauthenticated |
| `crapi/user/views.py` | `AdminUserView.create_token` — generates JWT via `jwt.encode(payload, secret, algorithm='HS256')` | **HIGH** — Symmetric signing (HS256); weak secret allows token forgery |
| `crapi/user/views.py` | `AdminUserView.verify_token` — validates JWT via `jwt.decode` with HS256 | **HIGH** — Vulnerable if secret is compromised; no expiration check visible |
| `runner.sh` | TLS enablement & cert/key passthrough to gunicorn; defaults to `certs/server.crt` / `certs/server.key` | **MEDIUM** — Cert/key paths configurable via env vars; no validation before gunicorn starts |
| `Dockerfile` | `COPY ./certs /app/certs` — copies host-side certs into container image | **LOW** — Standard build step; depends on host-side certs being present |
| `crapi_site/settings.py` | No `SECURE_SSL_REDIRECT`, `SESSION_COOKIE_SECURE`, or `CSRF_COOKIE_SECURE` settings | **MEDIUM** — Django TLS hardening settings absent; app may not enforce HTTPS-only cookies |
| `crapi/mechanic/views.py` | `MechanicCreateView.post()` — password hashing via `bcrypt.hashpw(password.encode("utf-8"), bcrypt.gensalt()).decode()` | **MEDIUM** — Production-facing password hashing; correct use of bcrypt with gensalt |
| `/core/management/commands/seed_database.py` | `seed_database()` — password hashing via `bcrypt.hashpw()` for DB seeding | **LOW** — Management command only; not exposed to end users |
| `/crapi/apps.py` | `SeedDataCommand.handle()` — password hashing via `bcrypt.hashpw()` on startup | **LOW** — Django management command; not production-facing |
| `/utils/helper.py` | `basic_auth(username, password)` — base64 encodes `username:password` for HTTP Basic Auth header | **HIGH** — NOT password hashing; base64 encoding of credentials is trivially reversible |
